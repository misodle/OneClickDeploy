What? This assistant produces the code for a filter that searches for information using a specified physical file or logical view.

Filters are used to dynamically create business object instance lists (e.g. lists of Customers, lists of Products, lists of Orders, lists of Employees, etc).

Typically Visual LANSA Framework filters are presented like this example (in the area circled in red):

 
Who? New Developers 
Where? Browser platforms (generates Visual LANSA Browser code (RDMLX))
How?
Click the Next >> button at the bottom of this form. 


Choose the file containing the data and the server module that is used to access it.

Then drag and drop the fields you want to load on the instance list.

(Drag the fields you want to show on the list, and also the unique identifier fields. )

(If you want the instance list to be changed to match your selection, check the checkbox.) 
Click the Next >> button. 
Choose the physical file or logical view that will be searched by the filter and set the search options and limits you want to use.

Click the Next >> button.


Check any other options you want.
Click the Generate Code button to generate the filter code. 
Enter a name and description and click the Create button to directly create a new reusable part from the generated code,

or,

Click the Copy Code to Clipboard button and then paste the source into a reusable part (with Target platform Web browser) that you have already created.
Note! If you need more details about any option while you are using this assistant click on the option and then use F1 to display the associated help text details.
