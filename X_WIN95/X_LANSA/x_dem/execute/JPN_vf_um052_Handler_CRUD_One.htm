機能 この CRUD ウィザードは、ビジネス オブジェクトのコマンド ハンドラーのためのコードを生成します。

ビジネス オブジェクトのコマンド ハンドラーは、選択された、ひとつ以上のビジネス オブジェクト インスタンス (例: 社員の表示、製品履歴の表示、選択されたすべてのオーダーの印刷など) に対して、コマンドを実行します。

Visual LANSA フレームワークでは、コマンド ハンドラーは多くの場合、以下のようにユーザーに表示されます (赤で囲んだ部分):

しかし、コマンド ハンドラーの表示方法は、他にもたくさんあります。コマンド ハンドラーの詳細と使用例については、ヘルプ アシスタントを参照してください。

  
対象者 新規の開発者
環境 ブラウザ プラットフォーム (Visual LANSA のブラウザ コード (RDMLX) を生成)
詳細 CRUD ウィザードは、共通コードやシステム定義テーブルに格納された情報の CRUD (Create(作成)/Read (読み込み)/Update(更新)/Delete(削除)) に必要となる、コマンド ハンドラーやフィルターを、ユーザーが素早く生成できるように設計されています。このようなタイプのテーブルは、多くの商用アプリケーション (例: 会社、 通貨、郵便番号、地域コードなど) で見られ、一般的には行が少なく、更新の度合いも低いです。CRUD ウィザードにより、この種のテーブルの保守を行うアプリケーションの構築がスピードアップされます。CRUD ウィザードを使用した推奨開発サイクルは、以下の通りです。
  • 例えば、コード テーブル という名前のアプリケーションを定義します。
  • 各共通コードまたはシステム定義テーブルに対し、以下を行います。

    => ビジネス オブジェクト (例：通貨、郵便番号) を定義します。

    => フィルタをひとつだけ定義します。

    => コマンドを、新規作成、コピー、明細及び削除のみ有効にします。

    => (必要に応じて) Windows 及びブラウザ プラットフォームで次の処理を繰り返します

    • CRUD フィルター ウィザードを使用して、ビジネス オブジェクト向けのフィルター プログラムを作成します。これをスナップインします。
    • このウィザードを使用して、単独のコマンド ハンドラー プログラムを作成します。この単独のコマンド ハンドラーを、新規作成、コピー、明細及び削除の各コマンドにスナップインします (つまり、4 つのコマンドすべてに、コマンド ハンドラーとして同じプログラムを使用します)。
