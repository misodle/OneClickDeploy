機能 このウィザードは、ビジネス オブジェクトに属する子データを管理するコマンド ハンドラーを提供します。

コマンド ハンドラーとは次のようなものです。(赤で囲った部分)

コマンド ハンドラーには、拡張可能な子データのリストが含まれ、ユーザーは、このリストからのエントリーを選択して、このエントリー データの保守が可能です。

次の例では、親ビジネス オブジェクトは社員（PSLMSTに基づく）で、子データは社員スキル（PSLSKLに基づく）です。コマンド ハンドラーは選択された社員の全スキルを表示します。
対象者 新規の開発者
環境 Windows プラットフォーム (RDMLXを生成)
使用方法
このフォームの下にある [次へ ] をクリックします。

親ビジネス オブジェクトとして、Akey と NKey を選択します。
(コマンド ハンドラーがインスタンス リストから受け取る情報)


[次へ] をクリックします。
子データの読み込みに必要なファイルおよびビューを選択し、子データ読み込みに使用する親キーをドラッグします。

ファイルのアクセスに使用するサービス モジュールも指定します。

[次へ] をクリックします。
様々な追加オプションから必要なものを選択します。(詳細については F1 のヘルプを参照)

[コード生成] をクリックし、フィルターのコードを生成します。
名前と記述を入力し、[生成] をクリックして生成されたコードから新規再利用可能パーツを直接作成する、

もしくは

[クリップボードへコピー] をクリックし、既に作成されている再利用可能パーツにソースコードを貼り付けます (ターゲット プラットフォームは Web ブラウザ) 。
次のステップ コード生成ウィザード パート2 を使用して、残りのリスト項目のためのコードを生成します。
注意 このアシスタントのオプションの詳細を見たい場合は、オプションをクリックし F1キーを押してヘルプアシスタントを表示します。
