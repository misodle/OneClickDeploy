What? This assistant produces a command handler that maintains child data belonging to the business object.

The command handler looks like this (in the area circled in red):

The command handler contains its own expandable list of child data, and when the user selects an entry from this list, they can maintain the data for that entry.

In the example shown, the parent business object is Employees (based on PSLMST), and the Child data is Employee skills (based on PSLSKL). The command handler shows all the skills for the selected employee.
Who? New Developers 
Where? Windows platforms (generates RDMLX)
How?
Click the Next >> button at the bottom of this form.

Choose the Akey and NKey fields for the parent business object.
(The information the command handler receives from the instance list)


Click the Next >> button.
Choose the file and view required to read the Child data, and drag the parent key used to read the child.

Also specify the Server Module you are going to use to access the file.

Click the Next >> button.
Choose from various additional options (Use F1 for more details).

Click the Generate Code button to generate the filter code.
Enter a name and description and click the Create button to directly create a new reusable part from the generated code,

or,

Click the Copy Code to Clipboard button and then paste the source into a reusable part (with Target platform Web browser) that you have already created.
Then! use the Part 2 code assistant to generate the remaining code for the list item.
Note! If you need more details about any option while you are using this assistant click on the option and then use F1 to display the associated help text details.
