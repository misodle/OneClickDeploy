What actions can end users perform with the business objects you defined?

The VLF was designed to support Object-Action interfaces.
This means that a user first selects the (business) object they want to work with, then they indicate what action they want to perform on it.

Actions should be described using end user terminology, definitely not IT terminology
Short and concise words should be are used to describe actions, because the (business) object being worked with is already known.

Often short verbs are used to describe actions, such as Copy, New, Edit, Delete, Print, Approve, Transfer, Reply and Renew.
Short nouns are also used to refer to things that are linked a business object. For example, Details, History, Charges, Claims,
Attachments, Schedule, Contacts, Documents and Images
