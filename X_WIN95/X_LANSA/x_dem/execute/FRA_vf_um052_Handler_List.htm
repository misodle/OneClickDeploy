Quoi ? Cet assistant génère le code d'un gestionnaire d'action qui maintient les données enfant de l'objet métier.

Le gestionnaire d'action ressemble à ceci (voir à l'intérieur du cercle rouge):

Le gestionnaire d'action contient sa propre liste de données enfant, et quand l'utilisateur sélectionne une entrée de cette liste, il peut maintenir les données affichées.

Dans l'exemple ci-dessous, l'objet métier parent est les Salariés (basé sur PSLMST), et les données enfant sont les formations des salariés (basées sur PSLSKL). Le gestionnaire d'action affiche toutes les formations du salarié choisi.
Qui ? Nouveaux développeurs 
Où ? Plateformes Windows (génère du code RDMLX)
Comment ?
Cliquez le bouton Suivant >> en bas de ce formulaire.

Choisissez les zones Akey et NKey pour l'objet métier parent.
(Les informations que le gestionnaire d'action reçoit de la liste d'instance)


Cliquez le bouton Suivant >>.
Choisissez le fichier, les vues et les clés nécessaires pour lire les données enfant.

Cliquez le bouton Suivant >>.
Choisissez les zones que vous souhaitez voir dans la liste du gestionnaire d'action, et les zones à modifier.

Cliquez le bouton Générer le code pour générer le code du gestionnaire d'action.
Entrez un nom et une description et Créer un nouveau composant VL à partir du code généré,
ou
Copier le code vers le presse-papier puis collez-le dans un composant VL que vous avez déjà créé.
