Quoi ? Cet assistant génère le code d'un gestionnaire d'action qui maintient les données enfant de l'objet métier.

Le gestionnaire d'action ressemble à ceci (voir à l'intérieur du cercle rouge) :

Le gestionnaire d'action contient sa propre liste de données enfant, et quand l'utilisateur sélectionne une entrée de cette liste, il peut maintenir les données de l'entrée.

Dans l'exemple ci-dessous, l'objet métier parent est les Salariés (basé sur PSLMST), et les données enfant sont les formations des salariés (basées sur PSLSKL). Le gestionnaire d'action affiche toutes les formations du salarié choisi.
Qui ? Nouveaux Développeurs 
Où ? Plateformes Windows (génère RDMLX)
Comment ?
Cliquer le bouton Suivant >> en bas de ce formulaire.

Choisissez les zones Akey et NKey pour l'objet métier parent.
(Les informations que le gestionnaire d'action reçoit de la liste d'instance)


Cliquer le bouton Suivant >>.
Choisissez fichier et vues nécessaires pour lire les données enfant, et faites glisser les clés du parent utilisées pour lire l'enfant.

Précisez également le module de Service que vous allez utiliser pour accéder au fichier.

Cliquer le bouton Suivant >>.
Choisir parmi les différentes options proposées (Utilisez l'assistant d'aide F1 pour plus d'informations).

Cliquer le bouton Générer le code pour générer le code du filtre.
Entrez un nom et une description et cliquer sur le bouton Créer pour créer directement un nouveau réutilisable à partir du code généré,

ou

Cliquer le bouton Copier le code vers le presse-papier puis collez-le dans un réutilisable (avec plateforme cible navigateur Web) que vous avez déjà créé.
Ensuite ! utilisez l'assistant de code Partie 2 pour générer le code restant pour l'élément de liste.
NB : Si vous souhaitez plus d'informations sur les options possibles quand vous utilisez cet assistant cliquez sur l'option et utilisez F1 pour afficher les informations de l'assistant d'aide.
