Quoi ? Cet assistant CLMS génère le code d'un gestionnaire d'action d'objet métier. 

Les gestionnaires d'action d'objet métier exécutent une action sur une ou plusieurs instances d'objet métier (exple. Afficher un Salarié, afficher l'Historique d'un Produit, Imprimer toutes les Commandes sélectionnées).

Qui ? Nouveaux développeurs 
Où ? Plateformes Windows (génère du code RDMLX)
Détails ? Les assistants CLMS ont été conçus pour vous aider à développer rapidement les filtres et les gestionnaires d'action nécessaires à la Création/Lecture/Modification/Suppression des informations stockées dans le code commun ou les tables de définition du système. Ces types de tables existent dans de nombreuses applications (Sociétés, Devises, Code postaux, etc...) et ont peu de lignes et une faible fréquence de mise à jour. Les assistants CLMS accélèrent le processus de création des applications de maintenance pour ces types de tables. Le cycle de développement recommandé avec ces assistants ressemble à ceci :
  • Définir une application Tables de codes (par exemple)
  • Pour chaque code commun ou table de définition système :

    => Définir un objet métier (exple. Devises, codes postaux)

    => Définir juste un filtre.

    => Activer seulement les actions Nouveau, Copier, Détails et Supprimer.

    => Répéter pour les plateformes Windows et Navigateur (suivant les besoins):

    • Créer un programme critère de recherche pour l'objet métier avec l'assistant de gestionnaire d'action CLMS. L'intégrer au cadre d'application.
    • Créer un seul programme gestionnaire d'action avec cet assistant. Intégrer le gestionnaire d'action pour chaque action Nouveau, Copier, Détails et Supprimer (i.e. le même programme est utilisé comme gestionnaire pour les quatre actions).
