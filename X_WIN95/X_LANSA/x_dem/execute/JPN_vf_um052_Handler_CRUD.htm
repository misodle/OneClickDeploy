機能 このCRUDウィザードは、ビジネス・オブジェクト・コマンド・ハンドラ用のコードを生成します。

ビジネス・オブジェクト・コマンド・ハンドラは、選択されたひとつ以上のビジネスオブジェクトインスタンス(例：社員を表示する、製品のの履歴を表示する、選択されたすべてのオーダーを印刷する)に対して、コマンドを実行します。

多くのVisual LANSAフレームワークでは、コマンド・ハンドラは、以下のように表示されます(赤で囲んだ部分)。

しかし、コマンド・ハンドラを表示する方法は、他にもたくさんあります。詳細及びコマンド・ハンドラの例については、ヘルプアシスタントを参照して下さい。
対象者 新規の開発者
環境 Windowsプラットフォーム(RDMLXコードを生成)
詳細 CRUDウィザードは、フィルタ及びコマンド・ハンドラの迅速な生成を手助けするように設計されています。コマンド・ハンドラは、共通コードやシステム定義テーブルに格納された情報のCRUD(Create/Read/Update/Delete)で必要になるものです。これらのタイプのテーブルは、多くの商用アプリケーション(例：会社、 通貨、郵便番号、地域、等)に存在し、一般には列が少なく、更新の度合いが低いです。CRUDウィザードは、これらのタイプのテーブルの保守を行うアプリケーションの構築過程をスピードアップさせます。CRUDウィザードを使用した推奨開発サイクルは、以下のようになります。
  • テーブル保守と呼ばれる(ような)アプリケーションを定義します。
  • 各共通コードまたはシステム定義テーブルの場合：

    => ビジネス・オブジェクト(例：通貨、郵便番号)を定義します。

    => フィルタをひとつだけ定義します。

    => コマンドを、新規作成、コピー、明細及び削除のみ有効にします。

    => Windows及びブラウザプラットフォームで繰り返します(必要に応じて)：

    • CRUD ﾌｨﾙﾀ ウィザードを使用してビジネス・オブジェクト向けのフィルタ・プログラムを作成し、スナップインします。
    • このウィザードを使用してシングル・コマンド・ハンドラ・プログラムを作成します。シングル・コマンド・ハンドラを、新規作成、コピー、明細及び削除の各コマンドにスナップインします(例：4つのコマンドすべてに、コマンド・ハンドラとして同じプログラムを使用します)。
使用方法
このフォームの下部にある次へ >>ボタンをクリックします。

その後、ビジネス・オブジェクトの識別プロトコルとビジネス・オブジェクトを支える物理ファイルを指定します。

ビジネス・オブジェクトの識別プロトコルの詳細については、ヘルプアシスタント (F1キー押下)を利用して下さい。

次へ >>ボタンをクリックします。

 
CRUDフォームに表示するフィールドを指定します。

次へ >>ボタンをクリックします。

 

 

 
コマンド・ハンドラがサポートするオプションのタイプを定義づける各種オプションから選択します。

このフォームで表示されるオプションの詳細については、ヘルプアシスタント (F1キー押下)を利用して下さい。

 
コード生成ボタンをクリックして、コマンド・ハンドラ・コードを生成します。

 
クリップボードへコードをコピーボタンを使用します。

 
生成したコードをVisual LANSAエディタ内のコマンド・ハンドラ・プログラムにペーストします。
注意 このアシスタントのオプションの詳細を見たい場合は、オプションをクリックし F1 キーを押してヘルプアシスタントを表示して下さい。
